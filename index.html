pythonでID/PASSWORDの管理

インストール

pitというモジュールを使うとできるみたいです。
こちら を参考にしました。
sudo pip install pit
依存関係でインストールされるpyYAMLのエラーが表示されますが、
特に 問題はない ようです。
設定

インタプリターから以下を実行すると、エディタが開きますので、
改行で一旦変更を加えて保存します。
はまりどころは、参考にしたサイト に記載の通りでした。
from pit import Pit
conf = Pit.get('label名', {'require':{'login':'your ID','passwd':'password'}})
確認

~/.pit/default.yamlに設定内容が保存されるので内容を確認します。
使い方
from pit import Pit
test = Pit.get('label名')
print test['login'],test['password']

blogの新規作成の自動化

blogをtinkererとGitHubで始めたので、
新規作成と更新を自動化しようと思いfabricのレシピ書いていたのですが、
virtualenv使っていて、workonの部分でエラーが出てはまったので備忘録。
  • はじめに書いたレシピ
def blog_create():
    local('workon sphinx')
    cd('blog_tinkerer/sources')
    local('sh ~/scripts/sh/blog_create.sh')
  • blog_create.sh
#!/bin/sh
vim `tinker -p \`date +%Y/%m/%d\` | awk '{print $5}' | sed -e "s/^'//g" -e "s/'$//g"`
  • エラーの内容は以下のような感じです
/bin/sh: workon: command not found

いろいろと調べた結果、ここのサイト を参考に修正したらうまくいきました。

  • 修正後のfabricレシピ
def blog_create():
    with prefix('source /usr/local/bin/virtualenvwrapper.sh'):
        with prefix('workon sphinx'):
            local('sh ~/scripts/sh/blog_create.sh')

ノート

cdでビルドするディレクトリまでfabricのレシピの中に書きたかったのですが、うまくいかなかったので、仕方なくshの中に処理を入れました。

  • blog_create.sh
#!/bin/sh
cd blog_tinkerer/sources
vim `tinker -p \`date +%Y/%m/%d\` | awk '{print $5}' | sed -e "s/^'//g" -e "s/'$//g"`

実行すると該当ファイルをvimで開くところまでやってくれるので便利です。

写真の位置情報を取得して、住所を表示する

  1. 写真から位置情報を抜き出す
  2. GoogleMapのAPIを利用して位置情報から住所を取得する

1. 写真から位置情報を抜き出す

google_map.py

# coding: utf-8

from googlemaps import GoogleMaps

def address_to_latlng(address):
    gmaps = GoogleMaps()
    lat,lng = gmaps.address_to_latlng(address)
    return lat, lng

def latlng_to_address(lat, lng):
    gmaps = GoogleMaps()
    destination = gmaps.latlng_to_address(lat, lng)
    return destination.encode('utf-8')

2. GoogleMapのAPIを利用して位置情報から住所を取得する

exif.py

#!/usr/bin/env python
# coding: utf-8

import sys
import os
from pexif import JpegFile
import google_map

argvs = sys.argv

ef = JpegFile.fromFile(argvs[1])
geo = ef.get_geo()

print google_map.latlng_to_address(geo[0], geo[1])
